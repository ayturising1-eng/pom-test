- Paket adının yalnızca “PLMR Version 11.2.zip” olması.
- Önceki update.text kayıtlarının silinmeden korunması.

YAPILANLAR
1. Güvenli kendi kullanıcı adını değiştirme akışı
- Yalnız system_admin rolü için change_own_username Edge Function aksiyonu eklendi.
- Yeni kullanıcı adı 3–32 karakter ve mevcut username biçim kuralıyla doğrulanır.
- Yeni kullanıcı adının tekrar yazılması zorunludur.
- Mevcut 4 haneli PIN zorunludur.
- Mevcut PIN, eski kullanıcı adı + PIN + PLMR_PIN_PEPPER ile türetilen parola kullanılarak Supabase Auth üzerinde doğrulanır.
- Yeni Auth parolası yeni kullanıcı adı + aynı PIN + aynı PLMR_PIN_PEPPER ile türetilir.
- Önce Auth parolası güncellenir.
- Ardından commit_self_username_change_v1 RPC fonksiyonu profiles.username, session_revoked_at ve activity_logs kaydını tek veritabanı transaction'ında tamamlar.
- Veritabanı commit'i başarısız olursa Auth parolası eski türetilmiş parolaya geri alınır.
- Eski ve yeni kullanıcı adı rate-limit bucket kayıtları best-effort temizlenir.
- Başarılı işlemden sonra global refresh token iptali çağrılır ve mevcut tarayıcı local sign-out ile giriş ekranına döner.
- Eski kullanıcı adı profiles tablosunda kalmadığı için login kimlik aramasında geçersiz olur.

2. Tüm cihazlardan çıkış
- Yalnız system_admin rolü için global_logout Edge Function aksiyonu eklendi.
- commit_global_logout_v1 RPC fonksiyonu session_revoked_at ve activity log kaydını aynı transaction içinde yazar.
- Edge Function mevcut access token ile Supabase Admin global sign-out çağrısı yapar.
- Global refresh token iptali başarısız olsa bile session_revoked_at eski access token'ların PLMR V2 güvenlik kontrollerinden geçmesini engeller; sonuçta açık warning döndürülür.
- Arayüzde ayrı ve açık onay checkbox'ı bulunan “Tüm Cihazlardan Çıkış Yap” dialog'u eklendi.
- Mevcut normal Çıkış butonu scope=local davranışıyla aynen korundu.

3. Arayüz
- Yönetim > Kullanıcılar alanına yalnız System Admin için “Hesap Güvenliği” kartı eklendi.
- “Kullanıcı Adımı Değiştir” dialog'u eklendi.
- “Tüm Cihazlardan Çıkış Yap” dialog'u eklendi.
- Türkçe ve İngilizce metinler eklendi.
- Güvenlik kontrolleri mevcut modal ve form yapısından izole CSS ile eklendi.

4. Auth bridge

ROLLBACK GÜVENLİĞİ
- Auth parolası güncellendikten sonra DB transaction başarısız olursa eski username-derived parola geri yüklenir.
